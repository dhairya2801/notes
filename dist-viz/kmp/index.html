Knuth-Morris-Pratt string matching: find all occurrences of a pattern in text in O(n+m) time by precomputing a failure function (LPS array) that tells the algorithm how far to shift on mismatch.

What is the KMP Algorithm?

The Knuth-Morris-Pratt (KMP) algorithm finds all occurrences of a pattern in a text. Unlike the naive approach which shifts the pattern by one position on each mismatch (up to O(n×m) comparisons), KMP achieves O(n+m) by never re-examining characters it has already matched.

The Key Insight: Failure Function (LPS Array)

KMP preprocesses the pattern to build a Longest Proper Prefix which is also a Suffix (LPS) array. For each position i, LPS[i] = length of the longest proper prefix of pattern[0..i] that is also a suffix.

The LPS is built by comparing the pattern against itself. A variable len tracks how many prefix characters currently match a suffix ending at the position we're analyzing. We try to extend this match by comparing pattern[len] with pattern[i]. On mismatch, we fall back using the LPS itself: len = LPS[len-1].

How the Search Works

Two pointers: i (text position) and j (pattern position).
Match: If text[i] == pattern[j], advance both. If j reaches the end, pattern found.
Mismatch: If j > 0, set j = LPS[j-1] (slide pattern forward). If j == 0, advance i.
The pointer i never moves backward — guaranteeing O(n) search.

Complexity

Preprocessing: O(m) • Search: O(n) • Space: O(m) • Total: O(n+m) vs naive O(n×m).
