Consistent Hashing — Interactive Visualizer

Explore how consistent hashing distributes keys across nodes on a hash ring. Add nodes, store key-value pairs, and watch how data migrates when the topology changes.

The Problem

With naive hashing (hash(key) % N), changing the number of nodes N causes nearly all keys to remap. If you have 1 million keys and go from 3 to 4 nodes, ~75% of keys move — an avalanche of cache misses and data transfers.

The Hash Ring

Consistent hashing places both nodes and keys on a circular hash space (0 to 232−1). Each key is assigned to the first node encountered going clockwise from the key's hash position. When a node joins or leaves, only the keys in the affected arc need to move — on average just K/N keys.

Virtual Nodes (Vnodes)

With few physical nodes, the ring can be unbalanced — one node might own a much larger arc than others. Virtual nodes solve this: each physical node gets multiple positions on the ring (e.g., 100–150). This spreads load more evenly. Try the slider below to see the effect on distribution.

Try It

Add a few nodes, put some keys, then add or remove a node and watch which keys migrate. Adjust the vnode count to see how it affects balance. The event log records every operation.
