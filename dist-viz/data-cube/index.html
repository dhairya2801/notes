What is a Data Cube?

In OLAP (Online Analytical Processing), data is modeled as a multidimensional cube. Each axis represents a dimension (e.g., Product, Region, Time), and each cell holds an aggregate measure (e.g., total revenue, average quantity). This enables fast, flexible analytical queries across any combination of dimensions.

Cube Operations

Slice — fix one dimension to a single value, reducing the cube by one dimension (3D → 2D).
Dice — filter on multiple dimensions simultaneously, selecting a sub-cube.
Roll-up — aggregate to a coarser level, collapsing a dimension (e.g., months → year).
Drill-down — expand to a finer level (e.g., year → months). Click any cell to drill down.

Why Cubes? (from DDIA)

Star and snowflake schemas in data warehouses store denormalized facts with dimension tables. Data cubes (materialized aggregates) pre-compute summaries across dimension combinations, making common analytical queries near-instant. The trade-off is flexibility — pre-computed cubes can't answer arbitrary queries, so most warehouses keep the raw data alongside cubes.
